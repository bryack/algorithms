## ⚠️ ВАЖНО: Сложность алгоритмов (Big O)

**Для каждой карточки и темы необходимо знать сложность операций!**

### Карточки и их сложности:

| Карточка | Операция | Сложность | Вопрос на собеседовании |
|----------|----------|-----------|------------------------|
| **Слайс vs Массив** | Доступ по индексу | O(1) | Почему доступ по индексу O(1)? |
| | Append (амортизированно) | O(1) | Когда append становится O(n)? |
| | Вставка в начало | O(n) | Почему вставка в начало O(n)? |
| **Map** | Доступ/вставка/удаление | O(1) | Почему map O(1)? Что такое хэш-функция? |
| | Худший случай | O(n) | Когда map деградирует до O(n)? |
| **Goroutines** | Переключение контекста | ~O(1) | Почему горутины легче потоков? |
| **Channels** | Отправка/получение | O(1) | Что происходит под капотом канала? |
| **Mutex** | Lock/Unlock | O(1)* | *но может быть blocking |
| **Context** | Отмена | O(1) | Как context распространяет отмену? |

### Структуры данных (что проходили):

| Структура | Операция | Сложность | Вопрос |
|-----------|----------|-----------|--------|
| **Стек (список)** | Push | O(1) | Почему O(1)? |
| | Pop | O(1) | |
| **Стек (массив)** | Push (аморт.) | O(1) | Что такое амортизированная сложность? |
| | Pop | O(1) | |
| **Очередь (кольцевой буфер)** | Enqueue | O(1) | Почему кольцевой буфер лучше списка? |
| | Dequeue | O(1) | |
| **Связный список** | Доступ по индексу | O(n) | Почему O(n)? |
| | Вставка в начало | O(1) | Почему O(1)? |
| | Вставка в конец (с tail) | O(1) | |
| | Вставка в середину | O(n) | |

### PostgreSQL сложности (ВАЖНО!):

| Операция | Сложность | Комментарий |
|----------|-----------|-------------|
| SELECT по primary key | O(1) | B-tree индекс |
| SELECT по индексу | O(log n) | Поиск по B-tree |
| SELECT без индекса | O(n) | Full table scan |
| JOIN | O(n×m) без индексов | Может быть очень медленным |
| INSERT | O(log n) | Обновление индексов |

## ⚡ НОВОЕ: Основы ЭВМ и ОС (критично для понимания Go)

По требованиям вакансии (High-Load, оптимизация) и вопросам на собеседованиях:

### Темы для изучения:

#### 1. Оперативная память (RAM)
- Стек (Stack) vs Куча (Heap)
- Выделение памяти в Go (escape analysis)
- Почему стек быстрее кучи

#### 2. Виртуальная память и указатели
- Что такое виртуальная память
- Страницы памяти (pages)
- Указатели в Go (указатель на стек vs указатель на кучу)
- Разыменование указателя

#### 3. Абстракции ОС (критично для понимания goroutines!)
- **Процесс** — что это, из чего состоит (адресное пространство, регистры, PCB)
- **Поток (Thread)** — чем отличается от процесса
- **Адресное пространство** — user space vs kernel space
- **Системный вызов** — что происходит при syscall
- **Контекст переключения** — почему дорого переключать потоки

#### 4. Кэш процессора (Cache)
- Уровни кэша (L1, L2, L3)
- Принцип локальности (временная и пространственная)
- Кэш-линия (обычно 64 байта)
- False sharing
- Почему связные списки хуже массивов для кэша
