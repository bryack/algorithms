# QWEN.md — Контекст проекта

## Обзор проекта

**Тип:** Учебный проект по алгоритмам и структурам данных  
**Язык:** Go 1.25.8  
**Модуль:** `github.com/bryack/algorithms`

Это персональный учебный репозиторий для изучения алгоритмов и структур данных с целью подготовки к техническим собеседованиям в крупных технологических компаниях (Big Tech).

Проект следует 6-месячному плану обучения (документирован в `Plan. Introduction to Algorithms.md`), который охватывает:
- Математические основы (логарифмы, прогрессии, комбинаторика)
- Паттерны на массивах и строках (Hash Map, Two Pointers, Sliding Window)
- Линейные структуры (стеки, очереди, связные списки)
- Рекурсия и Divide & Conquer
- Деревья и графы (BST, BFS, DFS)
- Динамическое программирование (мемоизация, табуляция)

**Важно:** Документация и комментарии в коде написаны преимущественно на русском языке.

## Соглашения о коде

### Стиль кода

- Следовать стандартным Go-конвенциям (`gofmt`, `golint`)
- Использовать осмысленные имена переменных на **английском** языке
- Включать комментарии с анализом сложности алгоритма:
  ```go
  // Time: O(n), Space: O(1)
  ```
- Предпочитать итеративные решения рекурсивным для случаев глубокой рекурсии (избегать stack overflow)
- Явно обрабатывать граничные случаи (пустые входы, одиночные элементы)

### Go-специфика для алгоритмов

1. **Строки — это байты:** Для Unicode-задач использовать `[]rune`, а не `[]byte`
2. **Слайсы передаются по значению, но разделяют underlying array:** Осторожно с side effects при модификации входных данных
3. **Порядок итерации по map рандомизирован:** Использовать слайс ключей, если важен порядок
4. **Нет встроенного Set:** Реализовывать через `map[int]struct{}` (empty struct не занимает память)
5. **Безопасность nil-указателей:** Всегда проверять на `nil` при работе со связными списками и деревьями

## Стратегия тестирования

- Писать unit-тесты для каждой реализации алгоритма
- Включать тест-кейсы для:
  - Нормальных случаев
  - Граничных случаев (пустой вход, одиночный элемент)
  - Граничных условий
  - Стресс-тестов производительности (где уместно)
- Файлы тестов следуют Go-конвенции: `<filename>_test.go`
