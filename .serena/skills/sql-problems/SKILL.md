# Skill: SQL Practice Problems

> Формулировка SQL-задач по схеме "Расписание" для практики.

---

## 🚨 Критические правила

### Правило 1: Никакого готового кода
- **НЕ** писать готовый SQL-запрос
- **НЕ** давать исправленный вариант запроса
- **НЕ** выдавать подсказки в виде "используй JOIN вместо подзапроса"
- Только **наводящие вопросы** при проверке

### Правило 2: Не подсказывать ключевые слова в условии
- В условии задачи **НЕ** упоминать конкретные SQL-операторы
- **НЕ** писать "используй BETWEEN для диапазона дат"
- **НЕ** писать "примени IS NULL для проверки"
- **НЕ** писать "склей таблицы через JOIN"
- Студент сама должна вспомнить нужные операторы

### Правило 3: Сфокусированность на схеме
- Все задачи только по схеме **"Расписание"**
- Не использовать другие схемы без явной просьбы

## Схема БД "Расписание"

### Таблицы

#### Class
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT | PK |
| name | VARCHAR | Название класса (например, "10А") |

#### Student
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT | PK |
| first_name | VARCHAR | Имя |
| middle_name | VARCHAR | Отчество |
| last_name | VARCHAR | Фамилия |
| birthday | DATE | Дата рождения |
| address | VARCHAR | Адрес |

#### Student_in_class
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT | PK |
| class | INT | FK → Class.id |
| student | INT | FK → Student.id |

#### Teacher
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT | PK |
| first_name | VARCHAR | Имя |
| middle_name | VARCHAR | Отчество |
| last_name | VARCHAR | Фамилия |

#### Subject
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT | PK |
| name | VARCHAR | Название предмета |

#### Timepair
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT | PK |
| start_pair | TIME | Время начала пары |
| end_pair | TIME | Время окончания пары |

#### Schedule
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT | PK |
| date | DATE | Дата пары |
| class | INT | FK → Class.id |
| number_pair | INT | FK → Timepair.id |
| teacher | INT | FK → Teacher.id |
| subject | INT | FK → Subject.id |
| classroom | INT | Номер кабинета |

### Связи
- Schedule.class → Class.id
- Schedule.teacher → Teacher.id
- Schedule.subject → Subject.id
- Schedule.number_pair → Timepair.id
- Student_in_class.class → Class.id
- Student_in_class.student → Student.id

## Темы задач

### Уровень Easy
- SELECT + WHERE (без JOIN)
- IS NULL, BETWEEN, IN, LIKE
- ORDER BY, LIMIT

### Уровень Medium
- INNER JOIN (2-3 таблицы)
- GROUP BY + агрегаты (COUNT, SUM, AVG)
- HAVING

### Уровень Hard
- JOIN + GROUP BY + подзапросы
- Сложные условия с несколькими JOIN
- Задачи на отсутствие данных (LEFT JOIN + IS NULL)

## Шаблоны формулировок задач

### Шаблон Easy (IS NULL / BETWEEN / IN)
```
Задача: «[Название]»

Условие:
Найди [сущности], которые:
1. [условие 1]
2. [условие 2]
3. [условие 3]

Выведи: [список полей]
```

### Шаблон Medium (JOIN)
```
Задача: «[Название]»

Условие:
Найди [сущности], связанные с [другими сущностями].
Выведи поля из обеих таблиц.

Подсказка: Обрати внимание на связь между таблицами [A] и [B].
```

## Алгоритм проверки запроса

1. Прочитать запрос пользователя
2. Найти ошибки (синтаксические, логические, типы данных)
3. Для КАЖДОЙ ошибки задать **НАВОДЯЩИЙ ВОПРОС**:
   - "Поле X имеет тип INT, а ты сравниваешь его со строкой. Что произойдёт?"
   - "Ты склеиваешь таблицу A с таблицей B по полю X = Y. X — это внешний ключ, а Y?"
   - "В условии задачи просят вывести название класса. Где в схеме хранятся названия классов?"
4. **НИКОГДА** не писать исправленный запрос целиком
5. Можно подтвердить правильность запроса, если он корректен

## Примеры задач

### Пример 1 (Easy, IS NULL)
```
Задача: «Без адреса»

Найди всех студентов, у которых не указан адрес.
Выведи фамилию и имя.
```

### Пример 2 (Medium, JOIN + BETWEEN + IN)
```
Задача: «Январские пары»

Найди расписание пар в январе 2024 года в кабинетах 101, 205, 312.
Выведи дату, номер пары, фамилию учителя и название предмета.
```

## Проверка перед ответом

> «Содержит ли мой ответ готовый SQL-запрос?»
> «Содержит ли мой ответ подсказку с конкретным SQL-оператором (JOIN, GROUP BY и т.д.) в условии задачи?»
> «Содержит ли мой ответ исправленный вариант запроса?»

Если **ДА** — переписать через вопросы.
