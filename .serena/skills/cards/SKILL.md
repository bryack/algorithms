# Skill: Работа с карточками

## Назначение

Определяет методику переработки карточек для подготовки к устным собеседованиям.

---

## Структура идеального ответа

```
[Простое определение своими словами] → 
[Объяснение механизма/принципа] → 
[Пример из проекта Task Manager/RAG] → 
[Практическая польза]
```

## Что НЕ делать

- ❌ **Ресторанные аналогии** — никаких "шеф-поваров", "официантов", "кухонь"
- ❌ **Код в ответе** — не писать `db := NewSQLite()`, это не произнести
- ❌ **Готовый длинный текст сразу** — пользователь пишет черновик сама
- ❌ **Книжные конструкции** — "Обеспечение легковесной конкурентности..."

## Что делать

- ✅ **Краткое определение** — 1-2 предложения
- ✅ **Разговорный стиль** — как на собеседовании
- ✅ **Связки** — "Проще говоря...", "То есть...", "По сути..."
- ✅ **Пример из её проекта** — Task Manager, RAG-сервис
- ✅ **Можно произнести на одном дыхании**

---

## Процесс работы

1. Пользователь показывает карточку или просит помощи
2. Я предлагаю варианты определений ИЛИ жду её формулировку
3. Она сама пишет черновик ответа
4. Я помогаю улучшить через наводящие вопросы, НЕ переписываю за неё
5. Добавляем пример из проекта в конце
6. Проверяем: можно ли сказать на одном дыхании?

---

## Пример переработки

### Было (книжный стиль)
> "Обеспечение легковесной конкурентности для одновременного выполнения множества функций в рамках одного процесса."

### Стало (разговорный стиль)
> "Горутины — это способ запускать задачи параллельно. Они очень легкие, всего 2KB памяти, поэтому можно запустить тысячи. Я в Task Manager использовал их для параллельной обработки запросов — вместо одного потока на запрос, получилось обрабатывать сотни одновременно."

---

## Наводящие вопросы для улучшения ответа

### Если определение слишком абстрактное
- "Как бы ты объяснила это коллеге за кофе?"
- "Что конкретно происходит при этом?"
- "Почему это лучше альтернатив?"

### Если нет примера из проекта
- "Где ты мог бы использовать это в Task Manager?"
- "А в RAG-сервисе это применимо?"
- "Приведи конкретную ситуацию, когда это спасло бы тебя."

### Если ответ слишком длинный
- "Как бы ты сказал это в 2 предложения?"
- "Какая главная мысль? Остальное — детали."
