# Skill: Работа с карточками

## Назначение

Определяет методику переработки карточек для подготовки к устным собеседованиям.

---

## Структура идеального ответа

```
[Простое определение своими словами] → 
[Объяснение механизма/принципа + Инженерные детали (Big O, структуры данных)] → 
[Пример из проекта Task Manager/RAG] → 
[Практическая польза]
```

## Что НЕ делать

- ❌ **Ресторанные аналогии** — никаких "шеф-поваров", "официантов", "кухонь"
- ❌ **Код в ответе** — не писать `db := NewSQLite()`, это не произнести
- ❌ **Готовый длинный текст сразу** — пользователь пишет черновик сама
- ❌ **Книжные конструкции** — "Обеспечение легковесной конкурентности..."

## Что делать

- ✅ **Краткое определение** — 1-2 предложения
- ✅ **Разговорный стиль** — как на собеседовании
- ✅ **Связки** — "Проще говоря...", "То есть...", "По сути..."
- ✅ **Пример из её проекта** — Task Manager, RAG-сервис
- ✅ **Можно произнести на одном дыхании**
- ✅ **Инженерная глубина** — упоминать Big O сложность и базовые структуры данных (стек, массив, мапа), если это применимо

---

## Процесс работы

1. Пользователь показывает карточку или просит помощи
2. Я предлагаю варианты определений ИЛИ жду её формулировку
3. Она сама пишет черновик ответа
4. Я помогаю улучшить через наводящие вопросы, НЕ переписываю за неё
5. Добавляем пример из проекта в конце
6. Проверяем: можно ли сказать на одном дыхании?

## Пример переработки

### Было (книжный стиль)
> "Обеспечение легковесной конкурентности для одновременного выполнения множества функций в рамках одного процесса."

### Стало (разговорный стиль)
> "Горутины — это способ запускать задачи параллельно. Они очень легкие, всего 2KB памяти, поэтому можно запустить тысячи. Я в Task Manager использовал их для параллельной обработки запросов — вместо одного потока на запрос, получилось обрабатывать сотни одновременно."

## Наводящие вопросы для улучшения ответа

### Если определение слишком абстрактное
- "Как бы ты объяснила это коллеге за кофе?"
- "Что конкретно происходит при этом (какая структура данных под капотом)?"
- "Какая алгоритмическая сложность (Big O) у этой операции?"
- "Почему это лучше альтернатив?"

### Если нет примера из проекта
- "Где ты мог бы использовать это в Task Manager?"
- "А в RAG-сервисе это применимо?"
- "Приведи конкретную ситуацию, когда это спасло бы тебя."

### Если ответ слишком длинный
- "Как бы ты сказал это в 2 предложения?"
- "Какая главная мысль? Остальное — детали."

## Вопросы для собеседования (Interview Questions)

### Назначение
Добавление в карточку типичных вопросов, которые могут задать на собеседовании backend-разработчику по этой теме. Цель — подготовка к стрессовым ситуациям, когда неожиданный вопрос может поставить в тупик.

### Процесс работы с вопросами

1. **Пользователь выбирает карточку** (например, "Виртуальная память")
2. **Я ищу вопросы** через:
   - Context7 (актуальная документация)
   - Интернет (поиск "[тема] interview questions backend developer")
3. **Я задаю 3-5 вопросов** по теме карточки
4. **Пользователь отвечает** или просит меня ответить (обучающий режим)
5. **Вопросы и ответы добавляются в карточку** в раздел "Вопросы для собеседования" в самом низу

### Формат вопросов в карточке

```markdown
## Вопросы для собеседования

Q: [Вопрос 1]
A: [Ответ 1]

Q: [Вопрос 2]
A: [Ответ 2]
```

### Типы вопросов (по приоритету)

1. **Поведенческие** — "Когда ты последний раз использовал X?"
2. **Практические** — "Как бы ты отладил проблему с Y?"
3. **Сравнительные** — "Чем X лучше Z?"
4. **Глубокие** — "Что происходит под капотом при X?"
5. **Edge cases** — "Что если X произойдёт при Y?"

### Что делать

- ✅ Искать актуальные вопросы для backend-разработчиков (Go, SQL, System Design)
- ✅ Задавать вопросы, которые проверяют понимание, а не зубрёжку
- ✅ Давать короткие, ёмкие ответы (30 секунд на произнесение)
- ✅ Связывать вопрос с проектом пользователя (Task Manager, RAG)

### Что НЕ делать

- ❌ Задавать вопросы уровня Senior для Junior позиции
- ❌ Слишком абстрактные вопросы без связи с практикой
- ❌ Длинные ответы, которые невозможно запомнить

## Работа с CSV-карточками

Карточки хранятся в корне проекта в файле `Программирование Карточки.csv` (формат `Front,Back`).

### Добавление / обновление карточки

Для безопасной перезаписи CSV используется Python-скрипт:

```bash
python3 .serena/skills/cards/scripts/update_csv_card.py '<Front>' '<Back>'
```

Что делает скрипт:
1. Читает CSV через `csv.reader`/`csv.writer` с `newline=''` (best practice из Context7 — иначе newlines внутри quoted fields и `\r\n` ломаются)
2. Ищет строку по **точному** совпадению `Front` (не substring)
3. Заменяет `Back` на новый текст
4. Пишет результат во временный файл и атомарно заменяет оригинал через `os.replace`
5. Если карточка не найдена — выводит `Card not found` и возвращает exit code `1`

### Способы передать Back

**Однострочный** — аргументом:
```bash
python3 .serena/skills/cards/scripts/update_csv_card.py "Название" "Однострочный ответ"
```

**Многострочный** — через stdin (рекомендуется):
```bash
python3 .serena/skills/cards/scripts/update_csv_card.py "Название" - << 'EOF'
Первая строка
Вторая строка
EOF
```

**Из файла**:
```bash
python3 .serena/skills/cards/scripts/update_csv_card.py "Название" --file back.txt
```

### Проверенные edge cases

- Back с запятыми — CSV writer автоматически оборачивает в кавычки
- Back с кавычками — экранируются удвоением `""`
- Back с `\n` и `\r\n` — корректно сохраняются внутри quoted field
- Пустой Back — записывается как пустое поле
- Большой Back (15+ KB) — работает без проблем
- Точное совпадение Front — не substring, соседние карточки не затрагиваются

### Альтернатива: inline Python

Если нужно сложное редактирование (дописать в конец, а не заменить полностью):

```python
python3 << 'EOF'
import csv
import os

filepath = "Программирование Карточки.csv"
temp_path = filepath + ".tmp"
target_front = "..."
questions_block = "\n\n---\n\n## Вопросы для собеседования\n..."

with open(filepath, newline='', encoding='utf-8') as infile, \
     open(temp_path, 'w', newline='', encoding='utf-8') as outfile:
    reader = csv.reader(infile)
    writer = csv.writer(outfile, quoting=csv.QUOTE_MINIMAL)
    for row in reader:
        if len(row) >= 2 and row[0] == target_front:
            row[1] = row[1] + questions_block
        writer.writerow(row)

os.replace(temp_path, filepath)
EOF
```
