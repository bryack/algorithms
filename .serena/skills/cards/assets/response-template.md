# Шаблон ответа на карточку

## Использование

Этот шаблон — ориентир для итоговой структуры ответа. НЕ копировать и вставлять — пользователь пишет сама.

---

## Шаблон

**[Термин]** — это [простое определение].

Проще говоря, [объяснение механизма своими словами]. Когда [ситуация], то [что происходит].

В [Task Manager/RAG-сервисе] я использовал [термин] для [конкретная задача]. Например, [детали ситуации].

Это позволяет [практическая польза]. Благодаря этому [результат].

---

## Пример заполнения (Generics)

**Generics** — это способ писать функции и структуры, которые работают с любыми типами данных.

Проще говоря, вместо того чтобы писать отдельные функции для `[]int` и `[]string`, пишешь одну с `[T any]` и она работает и с тем, и с другим. Когда вызываешь функцию, компилятор сам подставляет нужный тип.

В RAG-сервисе я использовал generics для HTTP-драйвера. Например, функция `post[T](url string, body any) (T, error)` принимает любой тип ответа и десериализует JSON прямо в него. Для метрик — `post[Metrics]`, для статуса — `post[Status]`.

Это позволяет не дублировать код для каждого типа. Благодаря этому уменьшилось количество boilerplate и ошибок при копировании-вставке.

## Подсказки для пользователя

Если пользователь застряла на каком-то блоке:

### Блок 1: Определение
- "Представь, что тебя спросили за кофе. Что это такое простыми словами?"
- "В 10 словах или меньше — что делает эта штука?"

### Блок 2: Механизм
- "Что происходит под капотом, когда ты используешь это?"
- "Как это работает технически?"

### Блок 3: Пример
- "Где в Task Manager это могло бы пригодиться?"
- "Приведи конкретную ситуацию, когда без этого было бы плохо."

### Блок 4: Польза
- "Зачем вообще это нужно?"
- "Что бы сломалось или было бы сложнее без этого?"
