# Проект: Task Manager

## Описание

Система управления задачами с параллельной обработкой.

## Архитектура

- **Язык**: Go
- **Конкурентность**: Goroutines + Channels + WaitGroup
- **Хранение**: SQLite (планировалось)
- **API**: HTTP REST

## Примеры использования в карточках

### Goroutines
> "В Task Manager использовал горутины для параллельной обработки нескольких задач одновременно — вместо того чтобы ждать последовательно, запускал каждую в своей горутине и собирал результаты через каналы."

### Channels
> "Использовал каналы для передачи результатов между горутинами. Например, одна горутина обрабатывала задачу, а через канал отправляла статус в основной поток для записи в БД."

### WaitGroup
> "При запуске пакета задач использовал WaitGroup, чтобы дождаться завершения всех горутин перед закрытием программы. Это гарантировало, что все задачи завершатся корректно."

### Context
> "Для graceful shutdown использовал context.WithCancel. При получении сигнала SIGINT отменял контекст, и все горутины получали сигнал завершиться корректно."

### Graceful Shutdown
> "В Task Manager реализовал graceful shutdown: при остановке программы ждал завершения текущих задач, не принимал новые, сохранял состояние."

### Dependency Injection
> "Структура TaskService принимала интерфейс Storage в конструкторе. Это позволяло тестировать сервис с моком хранилища, не поднимая реальную БД."

## Технические детали (для примеров)

- Параллельная обработка задач
- Очередь задач через каналы
- Graceful shutdown при SIGINT
- DI-контейнер для сервисов
- Unit-тесты с моками
