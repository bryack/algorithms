# Skill: Подготовка к алгоритмическому собеседованию в Яндекс

> Методология имитации реального алгоритмического собеседования в Яндекс на основе официальной инструкции: https://yandex.ru/jobs/interview/algorithms
>
> **Цель**: Научиться решать 2 задачи за 1 час в условиях, максимально приближённых к реальному интервью.

---

## 🎯 Философия

Это **не учебный трек** вроде NeetCode 150. Это тренировка реакции на собесе:
- Условие без примеров — как даёт интервьюер.
- Тесты придумываешь сама — это первый шаг, который оценивается.
- Код должен быть ясным и компактным — без «защиты от дурака», которую покрывает основной алгоритм.
- Время ограничено — 30 минут на задачу.

---

## 📋 Формат выдачи задачи

Ассистент выдаёт задачу по шаблону `assets/task-template.md`:
- **Название и сложность** — для ориентации.
- **Условие** — 2-3 предложения сути.
- **Что дано / Что нужно вернуть** — сигнатура функции.
- ❌ **Нет примеров входных/выходных данных** — как на реальном собесе.
- ❌ **Нет подсказок по алгоритму или структурам данных**.
- ❌ **Нет псевдокода**.

---

## 🔄 Pipeline решения (ожидается от пользователя)

```
1. Тестовые примеры  →  2. Алгоритм словами  →  3. Код  →  4. Мысленный прогон
```

### 1. Тестовые примеры (обязательно)
- Придумать 3-5 примеров: классический, пограничный, вырожденный.
- Показать интервьюеру (ассистенту) методичный подход.
- **Не начинать писать код до этого шага**.

### 2. Алгоритм словами
- Описать логику решения текстом.
- Определить структуры данных.
- Оценить сложность (Time / Space) до написания кода.

### 3. Код
- Писать в онлайн-редакторе (моделируем условия собеса).
- Не требуется запуск — компиляция мысленная.
- Go 1.25.8, стандартная библиотека.

### 4. Мысленный прогон
- Прогнать код на своих тестах из шага 1.
- Отследить значения ключевых переменных.
- Если нашла ошибку — объяснить и исправить (это плюс).

## ✅ Чек-лист оценки решения (проверяет ассистент)

После написания кода оцениваем по критериям Яндекса:

| # | Критерий | Вопрос для проверки |
|---|----------|---------------------|
| 1 | **Компактность** | Нет ли избыточных guard-clauses, которые покрывает основной цикл? |
| 2 | **Отсутствие дублирования** | Можно ли вынести повторяющуюся логику в функцию / унифицировать условие? |
| 3 | **Корректность** | Проходит ли мысленный прогон на всех тестах пользователя? |
| 4 | **Сложность** | Верно ли указана Time / Space? Есть ли более эффективный путь? |
| 5 | **Edge cases** | Обработаны ли они без «перегиба»? (см. пример `max_ones_sequence` в инструкции Яндекса) |
| 6 | **Готовность к собесу** | Можно ли написать такой код за 15-20 минут в стрессовых условиях? |

## 🚦 Шкала оценки

| Оценка | Название | Что означает |
|--------|----------|--------------|
| 🟢 | **Готово к Яндексу** | Код чистый, компактный, проходит тесты, сложность оптимальная. Можно показать на собесе. |
| 🟡 | **Нужна полировка** | Решение верное, но есть лишние проверки, можно упростить, или не хватает тестов. |
| 🔴 | **Нужна доработка** | Баги, неэффективное решение, плохая читаемость, или не продуман алгоритм до кода. |

## 📝 Правила для ассистента

1. **НЕ давать примеры входных/выходных данных** — пользователь придумывает сама.
2. **НЕ давать алгоритм словами** — только наводящие вопросы на этапе прояснения условия.
3. **После кода проверять по чек-листу** — сравнивать с критериями Яндекса.
4. **Если код избыточен** — показать более компактный аналог (как в инструкции: «Слишком сложный код» vs «Решение поэлегантнее»).
5. **Запрашивать мысленный прогон** — если пользователь не сделала шаг 4 сама.
6. **Тайминг** — напоминать о лимите 30 минут на задачу (но не прерывать жёстко).
