# Шаблон выдачи задачи

## Структура

```
Задача: [Название]

Условие: [2-3 предложения суть задачи]

Что дано: [входные данные]
Что нужно вернуть: [ожидаемый результат]

Пример:
Вход: [пример входа]
Выход: [пример выхода]
Объяснение: [кратко почему]
```

## Что НЕЛЬЗЯ включать

- ❌ Шаги решения ("Шаг 1: создай map...")
- ❌ Подсказки сразу
- ❌ Псевдокод
- ❌ Объяснение алгоритма до попытки решения
- ❌ Упоминание конкретных структур данных ("используй Hash Map")

## Пример ПРАВИЛЬНОЙ выдачи

```
Задача: Калькулятор дробей

Условие: программа для работы с обыкновенными дробями (4 операции).
Результат всегда должен быть сокращён.

Что дано: две дроби (числитель/знаменатель) и операция
Что нужно вернуть: результат как сокращённая дробь

Пример:
Вход: 1/2 + 1/3
Выход: 5/6
Объяснение: привели к общему знаменателю 6, сложили 3/6 + 2/6 = 5/6
```

## После выдачи задачи

Задать наводящие вопросы:
1. "Что дано на вход?"
2. "Что нужно вернуть?"
3. "Какое самое простое решение приходит в голову? (даже если медленное)"
4. "Какая сложность у этого решения?"
5. "Есть ли способ сделать быстрее?"

**НЕЛЬЗЯ**: дать ответы на эти вопросы без попытки пользователя.
