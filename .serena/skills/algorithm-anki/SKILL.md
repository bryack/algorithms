# Skill: Anki-карточки для алгоритмических задач

> Методология создания Anki-карточек по решённым алгоритмическим задачам для закрепления паттернов, механики и edge cases.

---

## Зачем это нужно

Решение задачи один раз — не гарантирует, что ты вспомнишь паттерн через месяц на собеседовании. Spaced repetition переводит «узнавание» в «воспроизведение». Одна задача → 3 короткие карточки. Никакого кода на карточках — только паттерны, шаги и граничные случаи.

---

## Структура: 3 карточки на задачу

Каждая задача, достигшая уровня понимания (L3+), разбивается на три карточки. Одна карточка — один факт.

### Type A — Pattern Recognition
**Проверяет:** узнаю ли паттерн по описанию задачи.

| Часть | Формат |
|-------|--------|
| **Лицо** | «Дан [входные данные]. Нужно [цель]. Какой подход оптимален и почему?» |
| **Тыл** | Название паттерна. Time / Space Complexity. Одно предложение — почему этот паттерн подходит. |

### Type B — Algorithm Mechanics
**Проверяет:** помню ли конкретные шаги реализации (не код, а логику).

| Часть | Формат |
|-------|--------|
| **Лицо** | «Задача X. В ситуации Y что происходит с [указателями / индексами / окном / счётчиком]?» |
| **Тыл** | Конкретное действие + условие, при котором оно выполняется. |

### Type C — Edge Case & Complexity
**Проверяет:** не ломается ли алгоритм на граничных входах.

| Часть | Формат |
|-------|--------|
| **Лицо** | «Задача X. Что если [edge case: пустой вход, отсутствие решения, дубликаты, все одинаковые элементы]?» |
| **Тыл** | Результат работы алгоритма + объяснение, почему сложность остаётся прежней. |

---

## Правила формулировки

- **KISS:** не более четырёх предложений на тыле карточки. Если больше — разбей на две.
- **Один факт:** одна карточка = один вопрос. Не смешивай паттерн и механику.
- **Без кода:** на карточке нет готового кода. Только словесное описание шагов и паттернов.
- **Язык:** русский с английскими терминами (Two Pointers, O(n), Prefix Sum, Sliding Window).
- **Cloze deletion:** для формул и фактов используй пропуски `{{c1::...}}`.

---

## Workflow добавления новой задачи

1. **Решить самостоятельно.** Карточка создаётся только после понимания алгоритма (не раньше).
2. **Отстранение.** Подождать 10–15 минут после решения.
3. **Составить 3 карточки** по шаблонам Type A, B, C.
4. **Проверить по чек-листу** (см. ниже).
5. **Добавить в Anki** с тегами: тема паттерна, источник (LeetCode/VK), уровень сложности.

---

## Организация колод

Используй **теги в Anki** вместо отдельных колод на каждую тему. Одна колода «Algorithms», теги для фильтрации:

- По паттерну: `two_pointers`, `sliding_window`, `prefix_sum`, `bucket_sort`
- По источнику: `leetcode`, `vk_education`
- По retention: `L1`, `L2`, `L3`, `L4`, `L5`
- По типу карточки: `pattern`, `mechanics`, `edge_case`

Повторение: карточки `L1` показываются чаще (Again → через 10 минут), `L4/L5` — реже.

---

## Анти-паттерны (чего не делать)

| ❌ Не делай | ✅ Делай |
|---|---|
| Писать код решения на тыле карточки | Описывать шаги словами |
| Смешивать Type A и Type B в одной карточке | Две отдельные карточки |
| Добавлять карточку на задачу, которая не понята | Сначала довести до понимания (L3+) |
| Создавать карточки на каждую строчку кода | Фокус на паттерн, механику и edge case |
| Использовать длинные описания условия задачи | Краткое описание входа и цели |

## Пример: 167. Two Sum II — Input Array Is Sorted

### Карточка A — Pattern Recognition

> **Лицо:**  
> Дан отсортированный массив. Нужно найти два числа, которые в сумме дают target. Какой подход оптимален и почему?
>
> **Тыл:**  
> Two Pointers (left/right). O(n) time, O(1) space. Массив отсортирован — можно двигать указатели внутрь в зависимости от суммы.

### Карточка B — Algorithm Mechanics

> **Лицо:**  
> Two Sum II. sum > target и sum < target — что делаем с указателями в каждом случае?
>
> **Тыл:**  
> sum > target → правый указатель влево (j--). sum < target → левый вправо (i++). sum == target → нашли ответ.

### Карточка C — Edge Case & Complexity

> **Лицо:**  
> Two Sum II. target меньше любого элемента массива. Что вернёт алгоритм?
>
> **Тыл:**  
> Указатели сойдутся, цикл завершится, вернётся nil/пусто. Сложность останется O(n).

## Чек-лист перед сохранением карточки

Задай себе вопросы:

- [ ] Содержит ли карточка готовый код?
- [ ] Больше ли четырёх предложений на тыле?
- [ ] Проверяет ли карточка воспроизведение, а не только узнавание?
- [ ] Один вопрос — одна карточка? Не смешаны ли типы?
- [ ] Понятен ли вопрос без контекста (можно ли ответить, не вспоминая условие целиком)?

Если хотя бы на один ответ «да» на первые два вопроса — переделать карточку.

---

## Повторение: что делать с оценками

- **Again** — не вспомнил паттерн / механику. После сессии Anki вернуться к задаче, перерешать на бумаге.
- **Good** — вспомнил с усилием. Нормальное состояние для закрепления.
- **Easy** — мгновенно. Карточка работает, интервал растёт.

Не ставь Easy просто так — если вспомнил «с натугой», это Good. Easy только при мгновенном ответе.
