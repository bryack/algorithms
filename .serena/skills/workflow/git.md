# Workflow: Работа с Git

## Критическое правило

**Я НЕ должен самостоятельно выполнять `git commit`!**

## Процесс

### Когда пользователь просит "Создай комментарий для git"

1. **Создать сообщение коммита** на английском языке
2. **Показать команды** `git add` и `git commit -m "..."`
3. **Спросить**: "Выполнить коммит или сделаешь сама?"

### Формат сообщения

```
[type]: [краткое описание]

[опциональное тело с деталями]
```

**Типы:**
- `feat:` — новая функциональность
- `fix:` — исправление бага
- `refactor:` — рефакторинг без изменения функциональности
- `test:` — добавление тестов
- `docs:` — документация
- `chore:` — рутинные задачи

### Что НЕЛЬЗЯ делать

- ❌ Не выполнять `git commit` без явного разрешения
- ❌ Не делать `git push`
- ❌ Не изменять историю git (rebase, reset и т.д.)

### Исключение

Если пользователь явно сказал **"сделай коммит"** или **"выполни коммит"** — тогда можно выполнить.

## Примеры

### Пример 1: Пользователь просит комментарий
```
Пользователь: Создай комментарий для git

Я: feat: add hexadecimal conversion with two's complement

Команды:
git add day_8/main.go day_8/main_test.go
git commit -m "feat: add hexadecimal conversion with two's complement"

Выполнить коммит или сделаешь сама?
```

### Пример 2: Пользователь просит выполнить
```
Пользователь: Сделай коммит

Я: [выполняю команды]
✅ Коммит создан: feat: add hexadecimal conversion...
```
