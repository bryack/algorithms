# Workflow: Восстановление после /compress

## Что такое compress

Команда `/compress` очищает контекст чата. Важная информация сохраняется в `.session_summary`.

---

## Процесс

### Перед compress

**Триггер**: "Создай сводку", "Сжатие", "Compress", подготовка к `/compress`

**Действия:**
1. Создать или обновить `.session_summary`
2. Включить ключевую информацию о сессии

**Содержимое сводки:**
```markdown
# Сводка сессии

## Дата
2026-03-23

## Выполненные задачи
- [x] Задача 1
- [x] Задача 2

## Текущая задача (в процессе)
- [ ] Название — что сделано, что осталось

## Статус обучения
- Фаза: 0 (Математический фундамент)
- Карточки: 10/54 переработано
- Следующая карточка: Clean Architecture

## Важные настройки
- [Критические правила, если изменились]

## Файлы
- [Изменённые/созданные файлы]
```

### После compress

**Триггер**: "Прочитай сводку", "Восстанови контекст", "Что мы делали"

**Действия:**
1. Прочитать `.session_summary`
2. Восстановить контекст из файла
3. Спросить, что пользователь хочет продолжить

---

## Пример диалога

### Перед compress
```
Пользователь: Сейчас сжатие, создай сводку

Я: [создаю .session_summary]

Сводка сохранена в .session_summary:
- Выполнено: LeetCode 405
- В процессе: карточка Generics (черновик готов)
- Следующее: Clean Architecture

Можешь вызывать /compress.
```

### После compress
```
Пользователь: Прочитай сводку

Я: [читаю .session_summary]

Восстановлено:
- Последняя задача: LeetCode 405 (завершена)
- Карточки: переработано 10/54
- Следующая карточка: Clean Architecture

Что будем делать? Продолжим карточки или перейдём к задаче?
```

---

## Файл .session_summary

Файл сохраняется на диске и переживает compress. Хранит:
- Статус задач
- Прогресс обучения
- Важные контекстные данные
- Ссылки на созданные файлы
