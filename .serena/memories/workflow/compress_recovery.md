# Workflow: Восстановление контекста после /compress (legacy)

## Процесс
1. **Перед `/compress`**: Пользователь просит создать сводку в `.session_summary`
2. **После `/compress`**: Пользователь просит прочитать `.session_summary` для восстановления контекста

## Действия ассистента
- При запросе сводки перед compress: создать/обновить `.session_summary` с ключевой информацией о сессии
- При запросе "прочитай сводку" после compress: прочитать `.session_summary` и восстановить контекст
- Файл `.session_summary` сохраняется на диске и переживает compress

## Содержимое сводки
- Дата и статус сессии
- Выполненные задачи
- Текущая задача (в процессе)
- Важные настройки и правила
- Версии инструментов
